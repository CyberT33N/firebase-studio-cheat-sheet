Dies ist eine √úbersicht √ºber die Konfigurationsoptionen in der `dev.nix`-Datei, die von Firebase Studio zur Anpassung deiner Entwicklungsumgebung verwendet wird.

## Top-Level Attribute

Diese Attribute befinden sich direkt im Haupt-Attributsatz, der von der Nix-Funktion zur√ºckgegeben wird.

### `channel`

*   **Zweck:** Definiert den Nixpkgs-Channel, der f√ºr die Paketverwaltung verwendet werden soll. Channels sind Sammlungen von Paketen in einem bestimmten Zustand (z.B. stabil oder aktuell).
*   **Type:** `String`
*   **M√∂gliche Werte:**
    *   `"stable-YY.MM"` (z.B. `"stable-24.05"`): Verwendet einen spezifischen stabilen Release-Zweig. Empfohlen f√ºr Konsistenz.
    *   `"unstable"`: Verwendet den neuesten Entwicklungszweig. Bietet die aktuellsten Pakete, kann aber weniger stabil sein.
*   **Beispiel:**
    ```nix
    channel = "stable-24.05";
    ```

### `packages`

*   **Zweck:** Liste der Softwarepakete, die in der Workspace-Umgebung installiert und verf√ºgbar gemacht werden sollen. Du kannst Pakete √ºber [Nix Package Search](https://search.nixos.org/packages) finden.
*   **Type:** `Liste von Nix-Paket-Derivationen` (typischerweise `pkgs.<paketname>`)
*   **Beispiel:**
    ```nix
    packages = [
      pkgs.nodejs_20 # Spezifische Node.js Version
      pkgs.zsh       # Z-Shell
      pkgs.starship  # Shell-Prompt-Anpassung
      pkgs.git
      pkgs.google-cloud-sdk
    ];
    ```

### `env`

*   **Zweck:** Definiert Umgebungsvariablen, die in der Workspace-Shell (Terminal) verf√ºgbar sein sollen.
*   **Type:** `Attributsatz` (Schl√ºssel-Wert-Paare, wobei Schl√ºssel und Werte Strings sind)
*   **Beispiel:**

### `idx`

*   **Zweck:** Ein verschachtelter Attributsatz, der Konfigurationen speziell f√ºr die Firebase Studio (IDX) IDE-Funktionen enth√§lt.
*   **Type:** `Attributsatz`

## `idx` Sub-Attribute

Diese Attribute befinden sich innerhalb des `idx`-Attributsatzes.

### `idx.extensions`

*   **Zweck:** Liste der VS Code-Erweiterungen, die automatisch im Workspace installiert werden sollen. Finde Erweiterungen und ihre IDs (Format: `publisher.extensionId`) auf dem [Open VSX Registry](https://open-vsx.org/).
*   **Type:** `Liste von Strings`
*   **Beispiel:**
    ```nix

      extensions = [
        "vscodevim.vim"           # Vim-Tastenbindungen
        "dbaeumer.vscode-eslint"  # ESLint Integration
        "esbenp.prettier-vscode"  # Prettier Code Formatter
    ```

### `idx.previews`

*   **Zweck:** Konfiguriert die Vorschau-Funktionalit√§t in Firebase Studio, z.B. f√ºr Webserver oder andere Dienste.
*   **Type:** `Attributsatz`
*   **Sub-Attribute:**

    *   `previews`: (Attributsatz) Definiert spezifische Vorschau-Konfigurationen. Der Schl√ºssel jedes Eintrags ist ein benutzerdefinierter Name f√ºr die Vorschau (z.B. `web`, `api`).

    *   `manager`: (String) Der Typ des Vorschau-Managers. `"web"` ist √ºblich f√ºr Web-Vorschauen im integrierten Browser-Panel.

    *   `rootDir`: (String, optional) Das Verzeichnis, von dem aus der `command` ausgef√ºhrt werden soll (relativ zum Workspace-Root).

*   **Beispiel:**
    ```nix

      previews = {
        enable = true;
        previews = {
          # Name der Vorschau: "web"
          web = {
            command = ["npm" "run" "dev"]; # Startet den Dev-Server via npm
            manager = "web";               # Nutzt die Web-Vorschau von IDX
            env = {

            };
          };

             # Alternativ: manager = "process"; # L√§uft nur als Hintergrundprozess ohne UI-Panel

               PORT = "$PORT";
               DATABASE_URL = "deine-db-url";
             };
             rootDir = "services/api"; # F√ºhrt den Befehl im Unterverzeichnis aus
          };
        };
      };
      # ... andere idx Optionen
    };
    ```
